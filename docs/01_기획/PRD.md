# Product Requirement Document (PRD)

## 1. 제품 개요 (Product Overview)
* **제품명:** Odab-Note (오답노트: AI Mistake Vaccine System)
* **슬로건:** "AI 에이전트의 고질적 실수를 사전 진단하고 치료하는 로컬 오답 백신"
* **핵심 가치:** 
  1. **보안성:** 에러 로그 및 프로젝트 코드를 유출하지 않고 로컬 SQLite DB에서 통제.
  2. **예방성:** LLM 모델의 고정적 지식 바이어스로 인한 반복 오류를 에이전트가 실행 전에 미리 '오답 프리셋' 형태로 주입받아 예방.
  3. **효율성:** 시간이 경과하면 서서히 망각(Decay)하는 가중치 시스템과 정규식 기반 정밀 매칭을 통한 AI 컨텍스트 최적화.

---

## 2. 해결하고자 하는 문제 (Problem Definition)
1. **AI 에이전트의 무한 컴파일 루프:** 에이전트가 코딩 작업 도중 컴파일 실패나 런타임 오류에 부딪히면, 방금 실행했던 잘못된 수정 방식을 또다시 적용해 무한 루프에 빠집니다.
2. **모델별 고유의 한계와 고질적인 실수:** 4B/8B 등의 경량 모델(온디바이스) 및 특정 상용 모델은 학습 셋의 한계로 인해 특정 모듈이나 버전을 다룰 때 항상 동일한 문법적/구조적 에러를 냅니다.
3. **컨텍스트(토큰) 오버헤드:** 과거의 모든 오답과 해법을 에이전트 시스템 프롬프트에 구겨 넣으면 토큰 소모가 극심하고, 에이전트의 집중력이 흐려져 또 다른 실수를 만듭니다.

---

## 3. 핵심 대상 고객 (User Persona)
1. **AI 에이전트 도입 기업 (Enterprise B2B):** 사내 보안을 위해 로컬 온디바이스 모델(8B, 14B)을 사내 개발망에 얹어 AI 에이전트를 구축하려 하나, 소형 모델의 저조한 코딩력과 삽질 때문에 골머리를 앓는 조직.
2. **독립 소프트웨어 개발자 (Pro Developers):** Antigravity, AutoGPT 등 에이전트 프레임워크를 상용 프로덕션 수준으로 굴리는 1인 개발자.
3. **AI 시뮬레이터 연구원 (AI Researchers):** AI 모델의 코딩 약점 데이터(Weakness Dataset)를 채굴하고 미세조정(Fine-Tuning) 셋을 확보하고자 하는 연구실.

---

## 4. 핵심 기능 요구사항 (Functional Requirements)

### F-01: 정밀 정규식 에러 매칭 엔진 (Regex Matcher)
* 에이전트가 마주한 Stack Trace 전체 텍스트를 인자로 받아, 오답 DB 내의 `error_pattern` 정규식들과 매칭하여 연관 솔루션을 추출.
* 대소문자 무시(case-insensitive), 줄바꿈 대응(DOTALL) 매칭 지원.

### F-02: 시간 감쇠(Time Decay) 가중치 알고리즘
* 오랜 기간 유발되지 않은 오답노트는 사용자의 토큰 최적화를 위해 가중치(`occurrence_count`)를 서서히 차감.
* CLI 및 스케줄러를 통한 자동/수동 감쇠 기능 구현.

### F-03: 모델별 맞춤형 프리셋 (Model-Specific Presets)
* 각 오답 레코드에 `target_model` 속성을 부여.
* `odab-note pull --model <model_name>` 실행 시, 사전에 내부 시뮬레이터로 마이닝된 암호화 유료 오답 프리셋을 로컬로 수집하여 SQLite DB에 머지.

### F-04: 오답 관계성 시각화 CLI (`odab-note graph`)
* 오답 간의 인과(Triggers) 및 충돌(Conflicts)을 ASCII 형태의 텍스트 네트워크 그래프로 터미널에 렌더링.

### F-05: 크라우드소싱 오답 공유 플라이휠 (Crowdsourced Telemetry)
* **익명 텔레메트리 연동:** 무료(Free) 사용자는 오답노트에 기록된 에러와 솔루션 데이터를 당사의 백신 수집 서버로 익명 업로드하도록 강제(의무화).
* **데이터 플라이휠 구축:** 크라우드소싱으로 매일 업로드되는 에러 로그를 분석하여 "최신 오답 백신 프리셋"으로 패키징하고, 이를 유료 티어 고객에게 유통하는 선순환 구조 구축.

---

## 5. 비기능 요구사항 (Non-Functional Requirements)
* **보안:** 로컬 데이터베이스 파일은 사용자 홈 디렉토리 하위의 안전한 저장소에 보관. 엔터프라이즈 티어에서는 무단 복제 및 유출 방지를 위해 **SQLCipher 기반의 통째 데이터베이스 파일 암호화(256-bit AES)** 적용.
* **성능:** 정규식 매칭 및 DB 조회가 에이전트의 턴당 100ms 이내에 완료되어야 함.
* **이식성:** `pip install` 및 Python 3.10+ 표준 환경에서 다른 서드파티 라이브러리 의존성을 최소화하여 작동.
* **Telemetry Control:** 무료 에디션은 텔레메트리 해제가 불가능하도록 CLI 소스 레벨에서 잠금 처리. 유료 에디션에서만 옵트아웃 옵션 제공.
